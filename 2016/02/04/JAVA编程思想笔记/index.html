JAVA编程思想笔记

  • JIT
  • Java

内存

  1. 一旦垃圾回收器准备释放对象，会先调用其finalize()，并在下次垃圾回收动作发生时，才真正回收对象。不建议在finalize()中手动清理。
  2. 只有在面临内存耗尽时，才会执行垃圾回收，因为这需要耗时。
  3. JAVA的对象是紧凑排列在堆上的（因此，每次释放对象后，都需要重排列堆中未被释放的对象），分配新对象时，其“堆指针”只需简单向前移动一格（类似于栈，所以很高效），而C++则需要在堆上查找空间。
  4. 两种内存回收机制：
    • 停止-复制（暂停程序，复制存活的对象到另一个堆）
    • 标记-清理（由栈区和静态存储区出发，遍历所有引用，标记对象，全部标记完后才开始清理动作）
    • 【前者不管是否有垃圾产生都会复制，效率低；后者清理垃圾之后，堆空间不连续，需重新整理。虚拟机会根据情况自动调整回收方式。】

效率

  1. JIT（Just-In-Time）可将程序部分或全部翻译成本地机器码（这本来是虚拟机的工作），使程序运行速度提升。

初始化

  1. 类内的基本数据成员保证都会有一个初始值（除boolean为false，其他都为0），对象引用为null。
  2. 类内变量的定义顺序决定初始化顺序。新建对象时：静态成员 => 非静态成员 => 构造函数

多态

  1. 静态方法不具有多态性。
  2. 组合技术通常用于想在新类中使用现有类的功能而非它的接口。
  3. 组合or继承？依据是：是否需要向基类向上转型。
  4. 组合一般是将现有类型作为新类型底层实现的一部分来加以复用，而继承复用的是接口。

陷阱

  1. 若子类覆盖了父类的方法，在父类中调该同名方法时，被调用的将是子类中的那个，且此时子类变量并未初始化（是0）。

接口

  1. 使用接口的核心原因：为了能够向上转型为多个基类型。
  2. 如果知道某事物应该成为一个基类，那个第一选择应该是使他成为一个接口。
  3. 当实现某个接口时，并不需要实现嵌套在其内部的任何接口。
  4. 使用原则：优先选择类而不是接口。从类开始，如果接口的必须行变得非常明确，那么久进行重构。

其他

  1. 每个源文件只能有一个public类，切类名要与文件名相同，否则报错。
  2. 使用jar文件是要指定具体路径和文件名，而不只是所在目录。
  3. 指定构造函数为private，可阻止此类继承。
  4. final修饰的基本类型，可以在编译时执行计算，减轻运行时负担。
  5. 对于基本数据类型，final使数值恒定不变；对于对象引用，final使引用恒定不变（数组也是对象）。
  6. 不能因为某数据是final的就认为在编译时可以知道它的值。
  7. 必须在域的定义出或者每个构造器中用表达式对final进行赋值。
  8. 类中所有的private方法都隐式地指定为是final的。
  9. 现代Java的容器库用ArrayList替代了Vector。
  10. 现代容器库用HashMap替代了Hashtable。
  11. static对象和static代码段都会在加载时，根据定义顺序初始化。
  12. Java中处了static方法和final方法（private方法属于final方法）之外，其他方法都是后期绑定。
  13. 声明方法为final可以“关闭”动态绑定。

设计模式

  1. 策略模式：一个能根据所传的参数对象的不同而具有不同行为的方法。

内部类

  1. 必须使用外部类的对象创建内部类：Outer.Inner inner = outer.new Inner();
  2. 如果定义一个匿名内部类，并希望它使用一个在其外部定义的对象，那么编译器会要求其参数引用是final的。
  3. 通过实例初始化，就能够达到为匿名内部类创建一个构造器的效果：
    public static Inner getInner(){
    return new Inner(){
    {print("Inside instance initalizer")}
}
    }
  4. 匿名内部类只能继承一个类或实现一个接口中的其中一个。
  5. 如果不需要内部类对象与其外围类对象之间有联系，那么可以将内部类声明为static，使其成为嵌套类。
  6. 普通类的不能哟static数据和static字段，也不能包含嵌套类。
  7. 可以通过多个内部类继承多个非接口类（类或抽象类），以实现“多继承”。
  8. 回调的价值在于它的灵活性，可以在运行时动态地决定需要调用什么方法。
  9. 一个类继承自内部类而非外围类时，默认的构造器并不算好，而且不能只是传递一个纸箱外围类的对象的引用。此外，必须在构造器内使用这样的语法：enclosingClassReference.super()。这样才提供了必要的引用，程序才能编译通过。
  10. “覆盖”内部类就好像它外围类的一个方法，其实并不起什么作用。

容器

  1. 你应该常见一个具体类的对象，将其转型为对应点额接口，然后在其余的代码中都使用这个接口。
  2. 使用Arrays.asList()的输出，将其当做List时，其底层表示的是数组，不能调整尺寸。
  3. TreeSet按照比较结果升序保存对象，LinkedHashSet按照被调价的顺序保存对象。
  4. ArrayList随机访问快，中间插入或移除较慢。LinkedList随机访问较慢，中间插入或删除代价较低，优化了顺序访问。
  5. List查找或删除某个元素时，都使用equals()比较对象引用。（indexOf(),remove(),retainAll(),removeAll());
  6. 如果传递给toArray()的参数数组太小，其将创建一个具有合适尺寸的数组。
  7. 迭代器能够将遍历序列的操作与序列底层的结构分离。
  8. Iteratot只能向前移动，但ListIterator可以双向移动。
  9. HashSet所维护的顺序与TreeSet和LinkedHashSet都不同。HashSet使用的是散列函数，TreeSet底层是红-黑树，LinkedHashList也使用散列，但看起来他使用了链表来维护元素的出入顺序。
  10. 自动包装机制将会发生在不能使用基本类型的容器的操作中。
  11. Queue接口窄化了对LinkedList的方法的访问权限，以使得只有恰当的方法才可以使用。
  12. 如果你创建了任何实现Iterable的类，都可以将它用于foreach语句中。
  13. foreach语句可以用于数组或其他任何Iterable，但是这并不意味着数组肯定也是一个Iterable,而任何自动包装也不会自动发生。不存在任何从数组到Iterable的自动转换，必须手工执行转换。
  14. 当你有一个接口并需要另一个接口时，编写适配器就可以解决问题。

  15. Arrays.toList()产生的List对象不会使底层数组作为其物理实现。

    Integer[] ia = {1, 2, 3};
    List list1 = new ArrayList(Arrays.asList(ia)); // 不会改变原数组
    List list2 = Arrays.asList(ia); // 会改变原数组

  16. 容器不能持有基本类型，但是自动包装机制会自动转换。
  • Queue和栈的行为，由LinkedList提供支持。
  • HashMap用于快速访问；TreeMap保持‘键’始终处于排序状态，没有HashMap快；LinkedHashMap保持元素插入的顺序，但也能通过散列提供快速访问。
  • Set不接受重复元素；HashSet提供最快的查询速度；TreeSet保持元素处于排序状态；LinkedHsahSet以插入顺序保存元素。
  • 新程序中不应该使用过时的Vector，Hashtable，Stack。
  • 除了TreeSet，所有Set都有与Collection完全一样的接口。
  • List和Collection存在着明显的不同，尽管List所要求的方法都在Collection中。
  • 标记接口java.util.RandomAccess附着到了ArrayList上，而没有附着到LinkedList上。这为想要根据所是有的特定的List而动态修改其行为的算法提供了信息。

人生的第一次面试——腾讯一面

  • 个人生活
  • 面试经历

距离今天搞好一星期，上星期四（4月9号）很高兴被邀去到腾讯一轮面试！这也是人生的第一次工作面试！

先来说说笔试

身在韶关

在一面之前，那天3月28日下午16：18分，收到了腾讯在广州举行的实习生招聘，时间为3月29日下午14：00，由于身在异地而同样投了简历的另一个小伙伴又还没有收到邀请，便纠结着要不要独自去呢？到了晚上，小伙伴也收到了邀请啦！！！既然有伴了，就不必纠结了！马上定下29日早上的火车票，来一场说走就走的笔试吧！

来到广州

原以为时间会很充足，因为火车到达广州火车站的时间距离开考还有2小时，本想可以吃个饭再去考，但出站后发现已经超过12点半了，到麦当劳买个汉堡，直接奔赴考场，出来地铁站已经13：40了，加上人生地不熟，该往哪走呢？没办法！被摩的坑一下吧！最后来到华工，插几分钟就开考了，排队签到找好位置坐下等开考！

马上开考

来到考试时间，突然，监考人员拿着有一米长的试卷(⊙o⊙)…我，呆了！！拿到试卷一看，原来有2/5是职位介绍和填写说明！ヾ(≧O≦)〃嗷~
做题过程中，发现试题考得非常广！什么C++、JavaScript、iOS、Java、Android、网络。。而且不管哪个方向都是同一份试卷。。。（ps：旁边坐着两个中大的，吓得我填学校名的时候都忘了“韶”字怎么写了o(╯□╰)o 还好灵机一动，拿出裤袋的火车票，才填上了）

回到韶关

考完笔试，回来等结果。随便把做题中发现自己对C++容器的了解不足，做了补习。。。

我的一面

再到广州

4月7日，收到腾讯一轮面试的邀请，时间4月9日上午9：00！这次不犹豫了，8号晚上就买火车票到广州的同学宿舍住了一晚，等待9号早上直接到面试地点！

一面过程

9号早上，6点多起床出发，8点10多分到了通知的酒店门口，以为我来的很早了！！没想到以上二楼，已经好几个等待面试的人在等着了。。个个手里还拿着资料在看（就是一开始那张照片的场景），而我手上却只带了手机一部。不能浪费那么长时间啊！于是上网找面经的博客看（那天只剩几M的流量，还是2G网速，想象可知当时的痛苦W(￣_￣)W ）。看到其中一篇说大多数面试都会问类似的题，就是比如说给你一份存有1亿行关键字搜索记录的文件，其中有相同项，请找出搜索量前10个的关键字记录，解题方法就是先用映射，在哈希去重，然后排序找出前十。看完这篇文章我就中毒了，什么问题都往这个方向想。。
过了9点，排队领号码牌，过程挺久的，人多，不知道过了多长时间后，我领到了我的号码牌，进入待面试区，不一会儿便通知我上楼面试。
来到面试房间，进去一看，一个人！不是我想象的多人！！也好！提交我简历，一些交流后，开始问我问题！
（下面是我能回忆起来的问题，感觉自己当时并不紧张，但好像说话有点语无伦次。。。）
(W代表面试官，A为我自己）
问题一
W：一个子类继承了一个基类，同时子类中有一个和基类完全相同的函数（返回值类型，函数名，参数都一样），问new一个对象，调用这个同名函数，会调用哪个？
A：调用子类的。它是根据new的动态对象调用的（想到虚函数去了）
W：那C++是怎么实现这一点的？
A：指针（又想到虚函数去了，但没跟他说！！）
W：指针？
A：每个类会有一个空间存储类的内容，对象回到相应的空间查找函数并调用。
问题二
W：印象最深的哪门专业课程？
A：信号与系统（他一问，我脑子就想起这个，我通信工程专业的）。
然后问了我一个信号与系统里的一个忘了什么原理，和它的实现原理，有什么用之类的（总之我只答了，减少运算，优化了，其他没答上来，我对信号与系统也不熟++）
后面还问了我，哪门课程是你一开始没学好，后面才搞懂的？我答数据结构（这次要找自己会的）！
问题三
他看了看我的简历，然后
W：说说你用的这个UIL框架（一个Android的图片异步加载框架）
我说我的理解（略）
问题四
W：有做过连接网络的应用吗？怎么与服务器交换数据的？
A：有，做过一个安卓应用，用服务器的开发人员提供的接口，对数据进行操作的。
问题五
W：说说TCP跟UDP的区别？那些TCP做了，而UDP没做的？
A：TCP需要消耗的服务器资源比UDP多得多；采用TCP连接的程序，发送数据后会确认是否发送成功，而UDP发送之后，就不再管了。
W：你说了TCP跟UDP的三大区别的其中一个，还有知道其他吗？
A：抱歉！我只记得这个，这个印象最深了。
问题六
W：除了Qt知道其他做做图形界面的吗？ （我简历里有提到用Qt做的项目）
A：我知道有MFC但没用过！
W：那为什么用Qt，不用MFC？有试过直接调用系统API做图形吗？
A：Qt可以跨平台，这对以后的多平台发展是好的！而MFC只能在windows平台上运行！没有用过系统API。
问题七
W：在一个1000行的代码里，有一个bug，有一个变量被赋值300，其中一次赋值导致了这个bug，问，除了打300个断点还有跟快的方式找出bug出现在哪次赋值吗?
A：像单元测试那样测试或者二分调试。
W：这样更慢。
A：用输出语句。
W：这样和打300个断点有什么区别？！
A：想不到其他了！
问题八
W：你知道什么排序算法，时间复杂度怎么样？
A：堆排序、归并排序、快速排序，据我了解公司一般用这几个，还有其他的插入排序、冒泡排序等，前三种排序时间复杂度都是nlg（n）。
（此时他在看手机，看了比较久，他问前面的问题的时候也会短暂看手机，最后发现原来他是在看我的笔试答题情况！因为看的比较久，我就接着答。。）
A：归并排序（忘了当时说的归并还是堆排序了，语无伦次了！）需要多一倍的空间来排序。
W：我发现今天你的面试跟你的笔试差别挺大啊！你对你的笔试满意吗？
A：不满意，考得广，有些像iOS、JavaScript那些都不会，还有C++的一些也没做出来，容器那些！
*问题九
W：你的笔试大题做的不好啊！出道题给你做做吧！比较反人类的！（代码如下） 

char *pStr=new char[10000000000]  <= e,d,g,c,y,g,d,h,...里面存了很多字符的主串  
char *pSubStr[n]  n个子串
        pSubStr[0] <= d,f,v...
        pSubStr[1] <= f.v.s...
        pSubStr[2] <= b,j,e...
        pSubStr[3] <= r.d.v...

问题：在主串中找出所有子串，并把它们去掉，不能使用string类，纯手写代码。
一拿到问题立马想到KMP，应该用KMP的思想吧！他没做声，然后我心想没那么简单吧？！不一会儿就往中毒的那篇blog说的方向想了，映射？哈希？排序？二分？然后我说完整代码我写不出，我说我的想法:
A：用哈希存子串。。。（略）
W：不行，哈希之后子串都被隐藏了而且也不知道去多长的主串去跟子串对比（长度这个我当时也想到了）
A：把子串建成二叉树，然后去配对
W：（笑）你现在不是在找问题的本质然后解决问题，而是在把你会的知识往题套，看能不能解出来！！
A：我做不出了！
W：这道题有人能在10分钟内写出60%的代码的，有这样的人的（我中间有问题时间限制问题）。好吧，我又超时了（面试我超时了。。），我们今天的面试就到这里吧！你有什么问题需要了解的吗？现在轮到你问我了！
A：进去之后工作分配是怎样的？
W：我是做在线教育的，只有用户装了qq就能通过在线视频学习。
A：那就是对数据压缩和传输方面的要求比较高咯？
W：是的。还有其他问题想了解吗？
A：没有了，谢谢！
W：好的，感谢你来面试！如果你适合我们，我们适合你，我们会在一个星期内通知你的！

就这样，面试就完了。

总结

面试完，想起之前师兄说过的：先要了解你要面试的公司在做什么（你学的那个方向），然后加强那方面的了解！从最后一个我问他的问题就印证了这个结论，在线视频教育，跟网络有关，所以问TCP和UDP，再就是图形界面。
在从酒店去往地铁站的路上，发现自己还是挺喜欢到陌生的世界的，这里没有人认识你，你也不认识其他人，有一种说不出的感觉，不用顾及那么多。

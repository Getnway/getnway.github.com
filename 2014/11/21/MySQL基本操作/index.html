MySQL基本操作

  • MySQL
  • 数据库

登陆与退出

登陆：

mysql -u root -p

退出：

quit 或 exit

数据库操作

新建数据库：

CREATE DATABASE [数据库名]

查看数据库：

SHOW DATABASES

进入数据库：

USE [数据库名]

数据表操作

新建表：

CREATE TABLE [表名] ( [列名] [类型] [约束], ....)

例如：

CREATE TABLE `ff` (                                             

      `id` int(9) NOT NULL AUTO_INCREMENT,                          

      `createtime` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,    

      `username` varchar(20) DEFAULT NULL,                          

      `record` varchar(20) DEFAULT NULL,                            

      `log` varchar(20) DEFAULT NULL,                               

      PRIMARY KEY (`id`)                                            

    ) ENGINE=InnoDB AUTO_INCREMENT=24 DEFAULT CHARSET=latin1

查看表：

SHOW TABLES

表结构：

DESCRIBE [表名]

删除表：

DROP TABLE [表名1],...

数据操作

增加：

INSERT INTO [表名] ( [列名1],... ) VALUES (' [字段1] ', ...)

删除：

DELETE FROM [表名] WHERE [列名1] = ' [字段1] ' ...

修正：

UPDATE [表名] SET [列名1] = "[字段1]

查询 ：

SELECT [列名1], ... FROM [表名] WHERE [列名1] = "[字段1]" ...

列操作

添加主键：

ALTER TABLE [表名] ADD CONSTRAINT PK_[表名] PRIMARY KEY ( [列名] )

增加列：

ALTER TABLE [表名] ADD COLUMN [新列名] [类型]

修改列属性： 

ALTER TABLE [表名] CHANGE [列名1] [列名2] [类型] [约束] ... 
ALTER TABLE [表名] MODIFY [列名1] [类型] [约束] ...

删除列：

ALTER TABLE [表名] DROP COLUMN [列名]

约束

自定义默认值：

DEFAULT [默认值]

自增：

AUTO_INCREMENT

主键：

PRIMARY KEY [列名]

外键：

CONSTRAINT [symol可默认产生] FOREIGN KEY ([列名]) REFERENCES [表名2]([列名2]) 
事件触发限制：[{ON DELETE | ON UPDATE} {RESTRICT | CASCADE | SET NULL | NO ACTION | SET DEFAULT}] 
主表删除或更新时，限制外表外键改动。跟随外键改动，设空值，无动作(默认)，设默认值

参考：http://www.cppblog.com/wolf/articles/69089.html
在MySQL 3.23.44版本后，InnoDB引擎类型的表支持了外键约束。

外键的使用条件：

  1. 两个表必须是InnoDB表，MyISAM表暂时不支持外键（据说以后的版本有可能支持，但至少目前不支持）；
  2. 外键列必须建立了索引，MySQL 4.1.2以后的版本在建立外键时会自动创建索引，但如果在较早的版本则需要显示建立；
  3. 外键关系的两个表的列必须是数据类型相似，也就是可以相互转换类型的列，比如int和tinyint可以，而int和char则不可以；

select now() as Systime;
select current_date as Systime;
select sysdate() as Systime;
